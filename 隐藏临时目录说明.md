# 隐藏临时目录功能

## 📋 修改说明

### 问题
`.temp_uploads` 目录是用于断点续传的临时上传目录，不应该在文件列表中显示给用户。

### 解决方案
在后端文件列表获取和搜索功能中，自动过滤所有以 `.` 开头的隐藏文件和文件夹。

---

## 🔧 代码修改

### 1. 修改 `file_manager.py` - 文件列表获取

**位置：** `backend/file_manager.py` 第 67-68 行

**修改内容：**
```python
for item in os.listdir(full_path):
    # 过滤掉隐藏文件和文件夹（以 . 开头）
    if item.startswith('.'):
        continue
```

**效果：**
- ✅ `.temp_uploads` 临时上传目录不再显示
- ✅ 其他系统隐藏文件（如 `.DS_Store`、`.git` 等）也不会显示
- ✅ 符合操作系统惯例（类 Unix 系统的隐藏文件约定）

---

### 2. 修改 `file_manager.py` - 搜索功能

**位置：** `backend/file_manager.py` 第 117-120 行

**修改内容：**
```python
# 使用 os.walk 遍历目录
for root, dirs, files in os.walk(full_path):
    # 过滤掉隐藏目录（以 . 开头）
    dirs[:] = [d for d in dirs if not d.startswith('.')]
```

**效果：**
- ✅ 搜索时不会搜索到隐藏目录
- ✅ 搜索结果中不会包含隐藏文件夹
- ✅ 提高搜索效率和准确性

## 🎯 影响范围

### 受影响的接口
1. **`GET /api/files`** - 文件列表接口
   - 返回的文件列表中不包含隐藏文件和文件夹
   
2. **`GET /api/search`** - 搜索接口
   - 搜索范围不包括隐藏目录
   - 搜索结果不显示隐藏文件

3. **`GET /api/folder-tree`** - 文件夹树接口
   - 间接影响（基于 `/api/files` 接口构建）
   - 移动对话框中的文件夹树不显示隐藏目录

### 不受影响的功能
- ✅ 文件上传功能（临时目录仍然正常工作）
- ✅ 断点续传功能（`.temp_uploads` 继续存储临时分片）
- ✅ 文件下载功能
- ✅ 文件重命名、删除、移动等操作

## 🔍 技术细节

### 为什么选择 `.` 作为标识？
1. **跨平台兼容** - Unix/Linux/macOS 系统都以 `.` 开头表示隐藏文件
2. **Windows 兼容** - Windows 也支持这种命名约定
3. **Git 兼容** - Git 仓库的 `.git` 目录也是此约定
4. **行业惯例** - 大多数文件管理工具都采用此约定

### 过滤实现
```python
# 列表推导式高效过滤
dirs[:] = [d for d in dirs if not d.startswith('.')]

# 注意：dirs[:] = ... 是原地修改，会影响 os.walk 的遍历
# 这是 Python 官方文档推荐的做法
```

## 📝 注意事项

### ⚠️ 重要提示
1. **不要手动访问隐藏目录** - 虽然前端不显示，但用户可以通过直接输入 URL 访问
   - 建议：后端也应该限制对隐藏目录的直接访问

2. **临时目录权限** - 确保 `.temp_uploads` 目录有正确的读写权限
   - 建议：设置适当的文件系统权限

3. **清理策略** - 定期清理旧的临时文件
   - 现有逻辑：`_cleanup_old_uploads()` 方法会清理超过 24 小时的未完成上传
